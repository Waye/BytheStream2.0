## 2026/4/20 — 批次 7：音频系统全线打通 + TTS 本地批处理

### 总览

今天把"音频"这块从纯占位推到了**端到端可用**：本地能生成 mp3、后端能查、前端能播、拖动进度条能跳位、未登录用户有默认欢迎音频。

整体架构（详细见 streaming.md）：
```
tts/ (MeloTTS 本地批生成) → output/*.mp3 + state.json
                                    ↓
        Fastify :4000 (audio.ts 读 state.json)
                                    ↓
        Expo App (expo-av Sound + Slider 拖动)
                                    ↑
        scripts/serve.py :8090 (Range 请求支持)
```

**关键参数**：

- MAX_CHARS_PER_CHUNK=400（拼接点少 → 电音机会少）
- 段落间静音 600ms

**简繁策略**：只生成简体 `_s` 的 mp3，繁体在后端 resolver 里 `_t → _s` canonical 归并。省一半时间和 50% 空间。

### MeloTTS 调试踩坑（按出现顺序）

1. **unidic 词典下载失败** — 网络在最后 1MB 卡死（S3 东京节点）
   - 解法：用已装的 `unidic-lite` 软链接到 `unidic/dicdir`

2. **NLTK cmudict 下载被拦** — SSRF 走 `198.18.0.18`（本机代理或 VPN 干扰）
   - 解法：手动 `curl` 从 GitHub raw 下三个 zip 到 `~/nltk_data/corpora/` 和 `~/nltk_data/taggers/`

3. **MeloTTS HParams 没有 `.get()` 方法**
   - 报错：`AttributeError: 'HParams' object has no attribute 'get'`
   - 解法：`synth.py` 里 `_pick_speaker` 用 `dict(spk2id)` 转换

4. **HuggingFace 模型下载极慢**（`/text-tools/melo-data/...`）
   - 解法：`export HF_ENDPOINT=https://hf-mirror.com`，每次新终端都要 export
   - 永久写入：`echo 'export HF_ENDPOINT=https://hf-mirror.com' >> ~/.zshrc`

5. **MPS device 不一致** — `expected scalar type Float but found Half`
   - 解法：`sed` 把 `melo/text/*_bert.py` 里 `device = "mps"` 全替换为 `pass # disabled`，强制 CPU

6. **state.json 一直为空** — `generate_one.py` 单篇脚本不写 state（只 batch.py 才写）
   - 解法：写了一个临时 Python 脚本，用 `ffprobe` 扫描 `output/volume_*/*.mp3` 补回 state.json

7. **后端 ESM 报错** `__dirname is not defined`
   - 原因：`server/` 是 `"type": "module"`，没有 CommonJS 的 `__dirname`
   - 解法：audio.ts 顶部 `const __dirname = path.dirname(fileURLToPath(import.meta.url))`

8. **Python 3.13 Homebrew externally-managed**
   - 报错：`× This environment is externally managed`
   - 解法：必须先 `conda activate python-mac-gpu`（不要用系统 Python 装包）

9. **brew cleanup truffle 权限报错** — 忽略，ffmpeg 已经装好可以用

10. **resolvers.ts 替换两次失败** — heredoc 单引号 + 中文括号 + 缩进差异
    - 解法：用更鲁棒的 `re.search` + 显式分隔符替换

### 性能优化
- **图片加速**：全局替换 `raw.githubusercontent.com` → `cdn.jsdelivr.net/gh/...`
  - 解决 Chrome CORB（Cross-Origin Read Blocking）批量阻塞（之前单页 ~40 次 `net::ERR_BLOCKED_BY_ORB`）
  - jsDelivr 有 CDN 边缘缓存，加载速度提升 5-10 倍
- **封面 URL 简化**：VolumeCard / CoverHero 不再猜 7 种文件名组合，直接信任后端 `coverImage` 字段
  - 每封面请求数：7 → 1
  - 首页 10 张封面总请求数：最多 70 → 10
- **Apollo 持久化缓存**：`apollo3-cache-persist` + AsyncStorage（key: `xsp_apollo_cache`，5MB 上限）
