### 用户认证（JWT + OAuth）
- 后端新增 `server/src/auth.ts`：JWT 签发/校验 + Google/Facebook token 验证
- 扩展 `UserDoc`：`provider` / `providerId` / `avatar` / `createdAt` / `updatedAt`
- GraphQL schema 新增：`AuthPayload` / `AuthProvider` enum
- 三种登录 mutation：
  - `loginWithGoogle(idToken)` — google-auth-library 验证
  - `loginWithFacebook(accessToken)` — debug_token + /me 两步验证
  - `loginOrRegister(email, name)` — 邮箱降级（开发/测试用）
- OAuth 账号自动绑定：优先 `(provider, providerId)` 匹配，未命中时按 email 绑定已有账号
- 前端 `lib/auth/oauth.ts`：`useGoogleAuth` / `useFacebookAuth` hooks（expo-auth-session）
- 未配 Client ID 时 hook 返回 stub，OAuth 按钮 disabled，不影响 email 登录

### Apollo Client 认证接入
- 改用 `@apollo/client/link/context` 的 `setContext`（官方 authLink）
  - 修复之前 `new ApolloLink` 里 `operation.setContext(async...)` 不等待 Promise 的 bug（token 永远来不及塞进 header）
- Token 存 AsyncStorage（key: `xsp_auth_token`），登录后自动带 `Authorization: Bearer`
- `bootstrapAuth` 启动时调 `me`，token 有效则自动恢复登录状态

### 文章收藏云同步

- Mutation：`addFavorite` / `removeFavorite`（upsert 防重复）
- Query：`myFavorites`（按加入时间倒序）
- 前端 `toggleFav` 逻辑：
  - 乐观更新本地 state（登录/未登录都立刻响应）
  - 登录后异步同步到后端，失败则回滚（重新拉 syncFavoritesFromServer）
  - 新收藏 unshift 到 favItems 前面（最新在前）
- 登录成功自动 `syncFavoritesFromServer` 拉回云端收藏
- 登出清本地 favItems（云端数据），保留本地音频收藏

### 性能优化
- **图片加速**：全局替换 `raw.githubusercontent.com` → `cdn.jsdelivr.net/gh/...`
  - 解决 Chrome CORB（Cross-Origin Read Blocking）批量阻塞（之前单页 ~40 次 `net::ERR_BLOCKED_BY_ORB`）
  - jsDelivr 有 CDN 边缘缓存，加载速度提升 5-10 倍
- **封面 URL 简化**：VolumeCard / CoverHero 不再猜 7 种文件名组合（`cover_s.png` / `{slug}.jpg` / `{num}_cover.png` 等），直接信任后端 `coverImage` 字段
  - 每封面请求数：7 → 1
  - 首页 10 张封面总请求数：最多 70 → 10
- **Apollo 持久化缓存**：`apollo3-cache-persist` + AsyncStorage（key: `xsp_apollo_cache`，5MB 上限）
  - 默认 `fetchPolicy: cache-first`（之前是 `cache-and-network`，每次都打网络）
  - 已访问页面二次打开 0ms，整页刷新 / 重启 app 后仍秒开
  - `me` / `myFavorites` 保持 `network-only`（必须实时）

### 修复的 bug
- ArticleCard ♥ 按钮传 `article.id`（string）→ store 判定为"移除"→ 但 Set 里没有 → no-op → 收藏永远加不上。改成传整个 `article` 对象。
- `.env` 文件末尾无换行导致 `echo >> .env` 把 JWT_SECRET 拼到 `CORS_ORIGINS=*` 后面，两个变量都坏了。用 `sed -i '' '/^CORS_ORIGINS=/d'` 清理后重新追加。
- Apollo authLink 用错 API：`new ApolloLink` 里 `operation.setContext(async cb)` 不会等待 Promise。改用 `setContext` 高阶函数（从 `@apollo/client/link/context` 导入）。
