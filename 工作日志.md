## 2026/4/18 — 图片系统 + UI 优化 + 内容页面

### 图片系统
- 全局替换 `react-native Image` → `expo-image`（三平台磁盘缓存）
- 文章卡片：后端返回 `firstImage`（从 content 提取第一张图），有图显示真实图片，无图 fallback 渐变+首字
- 期刊封面：后端返回 `coverImage`（从封面封底文章 content 提取），多 URL 智能匹配（`cover_s.png` / `{slug}.png` / `{num}_cover_s.jpg`），支持 png+jpg
- 期刊详情页 Hero 区：显示封面图右半（裁切双页封面），`onLoad` 后 reset urlIdx 避免首次白屏
- 文章内嵌图片：`AutoImage` 组件，`onLoad` 获取真实宽高，自适应显示竖图/横图
- 图片正则：支持 `<filename.jpg>` 和 `<filename.png, 作者>` 两种格式
- 后端聚合优化：85 次 `countDocuments` → 1 次 `aggregate`，期刊列表秒加载

### 首页优化
- 期号选择器：药丸横向滚动，最近 10 期可选，切换后文章滑窗自动更新

  - 第 4 页：士嘉堡堂地址（实景照片）→ Google Maps
- 底部 Footer：版权信息 + 法律声明 / Privacy Policy 链接 + 联系方式

### 全部期刊页

- 排序切换器：「最新优先」/「最早优先」，瞬间切换不闪烁

## 2026/4/17 — 批次 3&4: 后端 + 真实数据接入 + 简繁切换

### 后端 (Fastify + Mercurius GraphQL)
- `server/` 目录：独立 Node 项目，Fastify 替换 Express
- GraphQL schema：Article / Volume / SearchResult / User
- DataLoader 批量查询防 N+1
- 内存缓存（Redis 接口预留，配置 REDIS_URL 即可切换，TTL 1h）
- MongoDB Atlas 连接复用（现有 Xishuipang 数据库）
- 简繁过滤：slug 后缀 `_s` / `_t`，前端传 `character` 参数
- 搜索分页：`$text` 全文索引 + `offset/limit` 分页
- Heroku 部署配置就绪（Procfile），暂未部署

### 前端数据接入
- Apollo Client 3.x 接入，`_layout.tsx` 包 `ApolloProvider`
- 所有页面从 mock 切到 `useQuery` 真实数据
- `ContentItem.id` 从 `number` 改为 `string`（兼容 MongoDB slug）
- store 加 `character` 状态 + `toggleCharacter`
- store 加 `favItems[]` 数组（收藏列表不再依赖 mock）
- `toggleFav` 兼容 `string | ContentItem` 两种传参
- 首页最新文章 + 往期期刊改为水平滑动窗口
- 搜索页：每页 10 条 infinite scroll，显示「已显示 N / 总数 篇」

### 简繁切换（全局生效）
- 三个入口：文章阅读器工具栏 / 期刊详情按钮 / 用户中心设置
- Zustand store 全局 `character` 状态
